## Módulo 2 - Levantamento de Requisitos (Continuação)

- o que é Levantamento de Requisitos 
  -  Transformar as necessidades  de negócio em Informações técnicas 

- Requisitos Funcionais e Não Funcionais 
  -RF - O que o Sistema vai fazer
  - RNF - Como o Sistema vai fazer 

- Requisitos, Regras, e Restrções 
  - Requisitos:  A parte Técnica  do Sistema 
  - Regras: As necessidades dos Clientes 
  - Restrições: Limitações Impostas pela, linguagem, leis, normas,


-  Modelo de Documento de Requisitos 
|Campo|Conteúdo|Exemplo|
|----|------|------|
| ID | Código | RF-001 |
| Título | Valor | Tela de Login |
| Descrição | O Ator deve ser Capaz de | O Cliente conseguira faz Login |
| Prioridade | Valor da Prioridade | Alta |
| Regras Vinculadas | Outras Regras Relacionadas | RN-001 - Banco de Dados |

### Técnicas de Levantamento de Requisitos 
- Briefing
- BrainStorm
- Questionário
- Entrevista
- Entografia 
- Workshop
- Prototipagem 

#### Briefing
- Coleta de Dados Inicial
- Não Existe um Formato Pré-Definido
- Permite Coletas de Diversas Informações do Cliente
  - Manual de Identidade Visual (Cores, Logo, Slogan, Fontes ...)
  - Concorrentes
  - Prazos
  - Orçamento
  - Identidade da Empresa (Missões, Visão, Valores)
  - Público-Alvo

  #### BrainStorm
  - Reunião Criativa com Objetivo de Solucionar um ou mais **Problemas**
  - Podem ser Estruturados ou Não-Estruturados
  - Reunião Curta com Foco Bem Definido
  - Ferramentas de BrainStorm
    - Miro
    - Mentimeter
    - OneNote 

    #### Questionário 
    - Perguntas Direcionadas  ao Cliente, com Objetivo de Montar o Escopo do Projeto 
    (Documentação final do Projeto )
    - Formas de Fazer o Questionário
      - Reunião com Perguntas Diretas ( Qualitativas )
      - Formulários de Coleta de Dados ( Quantitativas )

      #### Entrevistas
      - Instrumento de Coleta de Dados com o Usuário do Projeto
      - Entrevistas podem ser Formais e Informais
      - Nessa Etapa é Necessário um MVP (Produto Mínimo Viável)

      #### Etnografia
      - Observação da cultura da Empresa, a Forma como ela Atende e se Relaciona com seu público Interno e Externo

      #### WorkShop
      - Técnica de Levantamento de Requisitos que acontecem na Apresentação de uma MVP, para um grupo Seleto de pessoas (StakeHolders) com Intuito de Absorver  a reação do Público ao Produto/Projeto
      - Eventos, Feiras e WorkShops de Produtos específicos

      ### Prototipagem
      - Criação de um Elemento de design para Simulação de um projeto , com intuito de Testar funcionalidades e elementos gráficos antes de começar a escrever Código
      - Os Protótipos podem ser de :
         - Baixa fidelidade (rafe) : Rascunho de um Desenho pode ser feito a Mão.
         - Média fidelidade (rough) : Desenhos no computador para Marcação de espaços e Layouts, não precisa de Texto nem de Cores.
         - Alta fidelidade : Protótipo focado nas Aplicações de Funcionalidade iguais aos do projeto que serão desenvolvidos posteriormente em Código.
